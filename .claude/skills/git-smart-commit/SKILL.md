# Git Smart Commit Skill
description: Automatically stages files and creates structured, atomic Git commits following conventional commit guidelines. Use this skill when the user asks to save progress or commit changes.

## Guidelines
- Check modified files using `git status`.
- Write a clean, brief commit message (e.g., "feat: add task description limits" or "fix: hide card scrollbar").
- Do NOT include long system paths or logs in the message.
- Execute `git add .` and `git commit -m "your message"`.
